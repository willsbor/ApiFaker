不過我那時候server端不太瞭解，而且我都用ObjectC code + define 來轉接api function 接口，以達到假資料的形態，但是最近感覺會有這個需求，1.接案子常常會遇到對方api已經開了大概，但是都沒有資料，（理論上對方提供假資料是不錯的做法，但是通常對方沒這個概念）2.視覺設計並沒有提供給設計師資料可能的大小，導致很多都沒有事先考慮到，例如：視覺設計指給了一張示意圖，上面只顯示了當content有10個字的時候該怎麼顯示，結果偏偏這個api吐出了30個字，工程師就用簡單的方法處理 ex: adjustTextWidth .... 結果，客戶大怒，字怎麼變小了，設計也說字不能變小呀，但是並不是程式做不到，只是一開始沒講好，所以倒霉的工程師只好默默地改方法。如果能事先先有些規範，應該可以避免大多數的誤會。
